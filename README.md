# Naura Landing page
# ⚠️⚠️ This is a public repository ! Never commit confidential or secret information here ⚠️⚠️

This repository contains the source code and configuration for the Naura website, which is automatically deployed using **GitHub Pages**.

## Overview

The website is hosted directly from this GitHub repository and published through GitHub Pages. Any changes merged into the main branch are automatically deployed and become available on the live website.

### Live Website
```text
https://nauralab.io/
```

---


## How It Works

The website was initially built using https://replit.com/. You must ask replit for a new version with static files only to be used on github pages.

### Deployment Flow

1. A contributor pushes changes to the repository main branch.
3. A GitHub Action workflow builds the website.
4. The generated site is published automatically.
5. The live website is updated with the latest changes.

This process eliminates the need for manual deployment steps.

---

## Updating the Website

### Editing Content

1. Create a new branch.
2. Make the required changes.
3. Commit your changes.
4. Open a Pull Request.
5. After review, merge the Pull Request into the main branch.
6. For faster deployments, since the code was generated by other tools, and if the code is already reviewed, you can commit directly to master to trigger the deploy.

Example:

```bash
git checkout -b update-homepage
git add .
git commit -m "Update homepage content"
git push origin update-homepage
```
Then open the Pull request, ask for a review, adn merge the pull request.

---

## Deployment Verification

After changes are merged:

1. Check the GitHub Actions workflow status.
2. Verify that the deployment completed successfully.
3. Open the live website and confirm that the changes are visible.

---

## Troubleshooting

### Changes Are Not Visible

* Wait a few minutes for deployment to complete.
* Clear browser cache.
* Verify that the deployment workflow succeeded.

### GitHub Pages Build Failed

* Review the build logs in GitHub Actions.
* Check for broken links or invalid configuration.
* Ensure all required files are committed.
